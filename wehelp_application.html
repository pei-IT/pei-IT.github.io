5請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

我想分享在中油服務期間，籌辦跨國視訊簽約儀式時的一次挑戰經驗。

情境：

當時我負責統籌一場重要的國際視訊簽約儀式，有來自三個國家、超過 40 位高層參與。這是公司的重大活動，需要協調頻寬升級、視聽設備設置、跨部門溝通等複雜工作。然而，在儀式前一天的測試中，發現視訊系統出現嚴重的音訊延遲問題，而且原因不明。

負面情緒：

當下我感到極大的壓力與焦慮。這是公司的重要活動，如果出錯會影響公司形象，而且牽涉到國際合作方。我感到挫敗、緊張，甚至開始自我懷疑是否有能力完成這個任務。

如何處理情緒與問題：

  1. 接受情緒，冷靜分析：我先讓自己冷靜下來，接受焦慮是正常的，但不能被情緒淹沒。我告訴自己：「問題已經發生，現在最重要的是解決它。」
  2. 系統化排除問題：我將問題拆解，逐一檢查網路頻寬、設備設定、軟體版本等可能因素。最終發現是某個會議室的舊設備與新系統不相容。
  3. 尋求支援：我立即聯繫設備供應商技術支援，並與IT團隊協作，連夜更換設備與重新設定。
  4. 準備備案：同時，我也準備了備用方案（使用其他會議室、備用設備），確保即使主方案失敗也有替代方案。
  5. 事後檢討與成長：活動順利完成後，我沒有只是鬆一口氣，而是詳細記錄了這次的問題與解決過程，並建立標準作業流程（SOP），避免日後重蹈覆轍。

學到的教訓：

  • 負面情緒是正常的，但要學會將它轉化為解決問題的動力
  • 系統化思考與準備備案能降低風險
  • 團隊協作與及時溝通非常重要
  • 每一次挑戰都是學習與成長的機會

6關於這份申請網頁，分享一個開發時的技術心得

技術選擇與考量：

  • 純 HTML/CSS 實作：我選擇使用純 HTML 和 CSS，而非引入複雜的框架。原因是這份網頁的核心目的是「清楚呈現內容」，而非展示技術複雜度。過度使用框架反而會增加載入時間與維護成本。
  • 響應式設計：使用 CSS 的 max-width 和 padding 確保網頁在不同裝置上都能良好顯示。我沒有使用 Bootstrap 等框架，而是手寫 CSS，這讓我更深入理解響應式設計的原理。
  • 視覺層次設計：使用漸層背景、陰影效果、邊框強調等視覺元素，讓內容層次分明。特別是使用藍色系配色呼應 WeHelp 的品牌形象，並使用圓形數字標籤讓問題編號更明顯。
  • 可讀性優先：選擇適合中文的字體、合適的行距（line-height: 1.6）、段落間距，確保長文本閱讀不疲勞。

技術細節：

  • 使用 CSS Grid 和 Flexbox 進行排版，確保內容對齊與間距一致
  • 使用 box-shadow 和 border-radius 創造現代感的卡片式設計
  • 使用漸層色（linear-gradient）讓視覺更豐富但不過度花俏
  • 注意色彩對比度，確保文字清晰可讀（符合 WCAG 無障礙標準）

7如何看待人工智慧的發展對軟體工程師的影響？

我認為人工智慧的發展對軟體工程師來說，既是挑戰也是機會，關鍵在於我們如何適應與運用這項技術。

正面影響：提升生產力與創造力

  • 自動化重複性工作：AI 工具（如 GitHub Copilot、ChatGPT）能協助生成樣板代碼、寫測試、除錯，讓工程師能專注於更有價值的架構設計與問題解決。
  • 降低學習門檻：新手可以更快獲得解答、學習最佳實踐，加速技能提升。我自己在利用 AWS 架設個人專案網站時就受益於 AI 輔助理解複雜的雲端架構。
  • 創造新的應用場景：AI 開啟了許多新的技術應用（如自然語言處理、電腦視覺），為工程師創造更多職涯發展方向。

挑戰：對基礎能力的要求更高

  • 不能只依賴 AI：AI 生成的代碼不一定正確或最佳化，工程師需要有足夠的基礎知識來判斷、修正與優化。
  • 差異化競爭力：當基礎的程式撰寫能力被 AI 輔助後，工程師的價值將更加體現在「系統思維」、「問題解決能力」、「架構設計」等更高層次的技能上。
  • 持續學習壓力：AI 工具本身也在快速演進，工程師需要不斷學習新工具與新技術。

我的應對策略：

  1. 扎實基礎，AI 為輔：我認為學習程式設計的基礎原理（資料結構、演算法、系統設計）比以往更重要。AI 可以幫我寫代碼，但它無法取代我對問題的理解與設計思維。
  2. 善用 AI 工具：我會積極學習使用 AI 輔助工具，但不依賴它。例如用 AI 幫助我快速原型開發、理解新技術，但最終的設計與決策仍由我來主導。
  3. 培養獨特價值：專注於 AI 較難取代的能力，如：跨領域知識整合（我的 IT 維運背景就是優勢）、溝通協作能力、商業需求理解、創新思維等。
  4. 擁抱變化：我相信軟體工程的本質是「解決問題」，而非「寫代碼」。只要保持學習熱情與問題解決能力，無論技術如何變化，都能找到自己的價值。

我認為 AI 不會取代優秀的軟體工程師，但會用 AI 的工程師會取代不會用 AI 的工程師。因此，我選擇積極擁抱這個變化，同時不斷強化自己的核心能力，讓自己成為能與 AI 協作、並創造更大價值的工程師。
